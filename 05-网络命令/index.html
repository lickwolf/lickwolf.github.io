    网络命令¶

    write

    命令所在路径：/usr/bin/write

    执行权限：所有用户

    语法：write <用户名>

    功能描述：给用户发送信息，以Ctrl+D保存结束

    eq：

    1.使用普通用户给root发送信息//用户一定要在线；查看用户在线的命令 w

    [xiaohan@centos7 ~]$ write root
woxiang genni yikuai chifan

    [root@centos7 ~]#
Message from root@centos7 (as xiaohan) on pts/0 at 19:32 ...
woxiang genni yikuai chifan
EOF

    挂载命令

    命令名称：mount

    命令路径：/bin/mount

    执行权限：所有用户

    功能：类似于windows中为新添加的设备分配盘符的过程；只不过这个过程是手动的

    语法： mount [-t 文件设备] 设备文件名 挂载点

    1.首先创建一个挂载目录

    [root@centos6 ~]# mkdir /mnt/cdrom

    2.执行挂载

    /dev/cdrom的软链接相当于 /dev/sr0//系统默认指定的外设就是/mnt/sr0

    //[root@centos6 ~]# ll /dev/cdrom lrwxrwxrwx. 1 root root 3 Oct 26 14:25 /dev/cdrom -> sr0

    其中 -t iso9660是可以省略的

    [root@centos6 ~]# mount -t iso9660 /dev/sr0 /mnt/cdrom/

    所以我们可以使用如下命令执行挂载

    //mount /dev/sr0 /mnt/cdrom

    使用mount命令将设备文件名和挂载点链接在一起

    //mount 设备文件名 挂载点

    提示没有写权限，为只读的。这是由于光盘文件的特性决定的；此时就已经挂载成功了

    //mount: block device /dev/sr0 is write-protected, mounting read-only

    [root@centos6 ~]# mount /dev/sr0 /mnt/cdrom/
mount: block device /dev/sr0 is write-protected, mounting read-only

    4.卸载挂载点//不能再挂载点下执行卸载

    在执行卸载的时候由于设备文件名/dev/sr0和 /mnt/cdrom之间已经建立好链接了，所以只需要卸载设备文件名/dev/sr0或者挂载点/mnt/cdrom都可以；但是当设备文件名和挂载点都写的话就会卸载两边，就会报错

    1.正常卸载

    [root@centos7 ~]# umount /dev/sr0
[root@centos7 ~]# ls /mnt/cdrom/
[root@centos7 ~]# ls -l
total 0

    2.在挂载点下进行卸载

    [root@centos6 cdrom]# umount /dev/sr0
umount: /mnt/cdrom: device is busy.
        (In some cases useful info about processes that use
         the device is found by lsof(8) or fuser(1))

    3.错误的卸载方式

    [root@centos6 ~]# umount /dev/sr0 /mnt/cdrom/
umount: /mnt/cdrom: not mounted
