    第一章 计算机系统概述¶

    1.1 计算机发展历程¶

    什么是计算机系统

    计算机系统 = 硬件 + 软件

    软件：

    • 系统软件：用来管理整个计算机系统（操作系统，数据库管理系统，网络软件，语言处理程序）
    • 应用软件：按任务需要编制成的各种程序

    硬件的发展

    发展阶段 时间 逻辑元件 速度（次/秒） 内存 外存
    第一代 1946-1957 电子管 几千-几万 汞延迟线，磁鼓 穿孔卡片,纸带
    第二代 1958-1964 晶体管 几万-几十万 磁芯存储器 磁带
    第三代 1964-1971 中小规模集成电路 几十万-几百万 半导体存储器 磁带，磁盘
    第四代 1972-现在 大规模，超大规模集成电路 上千万-万亿 半导体存储器 磁盘，磁带，光盘，半导体存储器。
    • 1947年，贝尔实验室，发明了“晶体管”
    • 1955年，肖克利在硅谷创建了肖克利实验室
    • 1957年，八叛徒创立仙童半导体公司
    • 1959年，仙童半导体发明了集成电路
    • 1968年，摩尔等人离开仙童，创立Intel
    • 1969年，仙童销售部负责人离开创立AMD

    1.2.1 计算机硬件的基本组成¶

    早期冯诺依曼的结构¶

    ​ “存储程序”的概念指的是讲指令以二进制代码的形式事先输入计算机的主存储器，然后按照其在存储器中的首地址执行程序的第一条指令，以后就按改程序规定的顺序执行其他指令表，直至程序执行结束。

    1.输入设备：将输入的数据和程序转换成计算机能够识别的形式，也就是0101这样的二进制数。

    2.存储器：存放数据和程序

    3.运算器：算术运算和逻辑运算

    4.输出设备：将运算结构转换成人们熟悉的形式

    5.控制器：用来协调其他组件相互协调和负责解析存储器中存储的程序指令；负责指挥程序运行

    在计算机系统上：软件和硬件在逻辑上是等效的

    Eg：对于乘法运算，可以设计一个专门的意昂见电路实现乘法运算，软件上的实现啊可以通过多次加法来实现。

    冯诺依曼计算机的特点：

    • 计算机由五大部件组成
    • 指令和数据以同等地位存于存储器，可以按地址寻访
    • 指令和数据用二进制表示
    • 指令由操作码和地址码组成
    • 存储程序
    • 以运算器为中心（输入/输出设备与存储器之间的数据传送通过运算器完成；导致运算的效率降低）

    现代计算机的结构¶

    现代计算机：以存储器为中心

    CPU = 运算器 + 控制器

    1.2.2 认识各个部件¶

    主存储器的基本组成

    存储体：数据在存储体内按地址存储

    MAR(Memory Address Register):存储地址寄存器

    MDR（Memory Data Register):存储数据寄存器

    存储单元：每个存储单元存放一串二进制代码

    存储字（world）：存储单元中二进制代码的组合

    存储字长：存储单元中二进制代码的位数（一般是bit的整数倍）

    存储元：即存储二进制的电子元件，每个存储元可以存储1bit

    每个存储单元会对应一个地址信息

    MDR位数＝存储字长

    Eg：

    MAR = 4位 → 总共只有2^4个存储单元

    MDR = 16位→ 每个存储单元可以存放16bit，1个字（world）= 16bit

    计算机的工作过程

    Eg:

    int a = 2, b= 3, c = 1, y = 0;
void main()
{
    y = a * b + c;

}

    a * b

    初:（PC） = 0,指向第一条指令的存储地址

    1:（PC）→MAR,导致（MAR）= 0；由控制器通过控制总线向主存储器指明此次进行读操作

    2: M(MAR)→M(MDR)，MAR通过主存地址找到存储单元中存储的二进制数据；并将这些二进制数据放置到MDR中；导致（MDR） = 000001 0000000101

    3：（MDR）-- >IR；MDR会将数据通过数据总线放置到IR（指令寄存器）当中。

    4：OP(IR)→CU(控制单元)，指令操作码（000001）会被送到CU（控制单元），CU经过分析后得知这是“取数”指令

    5：IR会将地址码（0000000110）放置到MAR，导致MAR = 6

    6：M（MAR）-- >MDR;；MAR会根据地址码从存储体当中找出主存地址为5的存储单元，将存储单元里的数据放置到MDR当中；导致（MDR）= 0000000000000010 = 3。

    7：MDR的数据会经过CU放置到ACC（累加寄存器）当中；至此完成“取数指令”

    8：PC的值会自动加1；指明下一条将要执行的指令

    b = 2

    初：（PC）= 1，PC的值经过上一次的加法运算使得pc = 1；PC指向贮存地址1

    1:（PC）→MAR,导致（MAR）= 1；由控制器通过控制总线向主存储器指明此次进行读操作

    2: M(MAR)→M(MDR)，MAR通过主存地址找到存储单元中存储的二进制数据；并将这些二进制数据放置到MDR中；导致（MDR） = 000100 0000000110

    3：（MDR）-- >IR；MDR会将数据通过数据总线放置到IR（指令寄存器）当中。

    4：OP(IR)→CU(控制单元)，指令操作码（000100）会被送到CU（控制单元），CU经过分析后得知这是“乘法”指令

    5：IR会将地址码（0000000110）放置到MAR，导致MAR = 6

    6：M（MAR）-- >MDR;；MAR会根据地址码从存储体当中找出主存地址为6的存储单元，将存储单元里的数据放置到MDR当中；导致（MDR）= 0000000000000011 = 3。

    7：(MDR)→CU;MDR的数据会经过CU放置到MQ（乘商寄存器）当中；

    8：（ACC）→X ;ACC会将a = 2 放到x寄存器当中

    9：（MQ)*(X) → (ACC);CU会通过控制总线通知ALU（算数逻辑计算单元）执行乘法运算;导致ACC = 6，如果乘积太大，则需要MQ辅助存储

    10：PC++；PC的值会自动加1；指明下一条将要执行的指令；此时PC指向主存地址2

    a * b + c

    初：（PC）= 1，PC的值经过上一次的加法运算使得PC = 2；PC指向主存地址2

    1:（PC）→MAR,导致（MAR）= 1；由控制器通过控制总线向主存储器指明此次进行读操作

    2：M(MAR) → M(MDR) ;MAR向存储体中找到主存地址为2的存储体，将其内部的数据读出来放置到MDR当中

    3：(MDR) → (IR);MDR中的数据通过数据总线传递给IR；

    4：(IR) → CU;IR将操作码（000011）发送给CU；CU对操作码进行分析得出这是”加法“指令

    5：（IR）→ MAR;IR将地址码发送（000000000111）到MAR，MAR将通过地址码找到主存地址为7的存储单元中的数据1；并且将1放置到MDR当中

    6：(MDR) -- > X;MDR通过数据总线将数据传送给X（通用寄存器）;导致X = 000000000001 = 1

    7：（ACC）+ X;CU会通过控制总线通知ALU（算数逻辑单元）执行加法运算；导致ACC = 7

    8：PC++;PC的值会自动加1；指向下一条需要执行的指令；此时PC指向主存地址3

    将结果存储到主存当中

    初：（PC）=2，PC的值经过上一次的加法运算使得PC = 3；PC指向主存地址3

    1：（PC）→ MAR;PC通过控制总线向主存储器指明此次进行读操作

    2：M(MAR) → MDR;MAR通过主存地址3读取存储单元中的数据 000010 00000001000放置到MDR当中；

    3：（MDR）→ IR; MDR将数据通过数据总线传送到IR当中

    4：（IR）→ CU;IR将操作码（000010）发送给CU；CU对操作码进行分析得出这是”存数“指令

    5：（IR）→ MAR;IR将地址发送给MAR；MAR读取到地址码（000000001000）;MAR = 8.

    6: （ACC）→ MDR;ACC将运算结果7存放到MDR当中

    7：CU(控制单元)会通知存储体此次进行一个写操作；将MDR中的数据存放到MAR指定的存储单元当中

    8：PC++;PC的值会自动加1；指向下一条需要执行的指令；此时PC指向主存地址4

    停机

    初：上一条指令执行完毕之后PC = 4

    1：（PC）→ MAR;MAR通过主存地址4将该存储单元中的数据放置到MDR当中

    2： （MDR）→ IR;MDR将数据通过数据总线发送给IR；

    3：（IR）→ CU;IR通过控制总线将将操作码发送给CU；经过CU分析得出改操作为停机操作

    存储器的性能指标

    总容量 = 存储单元的个数 x 存储字长 bit

    Eg:

    ​ MAR = 32 位；则有2^32次方个存储单元

    ​ MDR 为8位

    总容量 = 2^32 * 8bit = 4GB
